🏥 EHR AI Services

Database Schema & Model Relationships

🏢 Facility
Healthcare facilities (hospitals, clinics)
  • 🔑 id: UUID (Primary Key)
  • 📝 name: String (Required)
  • 📍 address: String
  • 📅 created_at: DateTime
  • 📅 updated_at: DateTime
  • 🔗 vector_db: One VectorDB
  • 🔗 users: Many Users
  • 🔗 patient_identifiers: Many PatientIdentifiers
🗄️ VectorDB
Vector database for AI embeddings per facility
  • 🔑 id: UUID (Primary Key)
  • 📝 name: String (Required)
  • 🔗 facility_id: UUID → Facility
  • 📅 created_at: DateTime
  • 📅 updated_at: DateTime
  • 🔗 facility: One Facility
  • 🔗 collections: Many Collections
📁 Collection
Document collections (patient-specific or shared)
  • 🔑 id: UUID (Primary Key)
  • 📝 name: String (Required)
  • 📄 description: String
  • 🔗 vector_db_id: UUID → VectorDB
  • 📅 created_at: DateTime
  • 📅 updated_at: DateTime
  • 🔗 vector_db: One VectorDB
  • 🔗 documents: Many Documents
👤 PatientIdentifier
Anonymized patient identifiers (HIPAA compliant)
  • 🔑 id: UUID (Primary Key)
  • 🏷️ patient_code: String (Unique, Required)
  • 🆔 external_id: String (Optional)
  • 🔗 facility_id: UUID → Facility
  • 📊 age_range: String (e.g., "25-30")
  • ⚧️ gender: String (Optional)
